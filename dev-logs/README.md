## 🎯 归档原则

### 文档分类标准
1. **按日期分组**：以月份为单位创建目录（如 `2025-11/`）
2. **按主题分类**：在月份目录下按功能模块或问题类型分类
3. **保留索引**：每个月份目录都有 `INDEX.md` 索引文件

### 命名规范
- 月份目录：`YYYY-MM/`（如 `2025-11/`）
- 主题目录：`YYYY-MM-DD-主题描述/`（如 `2025-11-15-导入功能优化/`）
- 索引文件：`INDEX.md`

## 🔍 如何查找文档

### 方法1：按时间查找
1. 确定大致的开发时间
2. 进入对应月份目录（如 `2025-11/`）
3. 查看该月的 `INDEX.md` 索引文件
4. 根据索引找到具体文档

### 方法2：按主题查找
1. 确定问题所属的功能模块
2. 在月份目录下找到对应的主题目录
3. 浏览该目录下的文档

### 方法3：全局搜索
```bash
# 在所有开发日志中搜索关键词

# 查找特定类型的文档
find dev-logs/ -name "*SUMMARY*.md"

find dev-logs/ -name "*GUIDE*.md"
```

## 📊 文档类型说明

- `*_SUMMARY.md` - 总结性文档
- `*_GUIDE.md` - 使用指南
- `*_FIX*.md` - 修复报告
- `*_IMPLEMENTATION*.md` - 实施报告
- `*_ANALYSIS.md` - 分析文档
- `*_REPORT.md` - 报告文档
- `QUICK_*.md` - 快速参考
- `INDEX.md` - 索引文件

## 💡 使用建议

1. **定期归档**：每次完成重要功能或修复后，及时归档相关文档

4. **命名规范**：使用清晰、描述性的文件名和目录名
