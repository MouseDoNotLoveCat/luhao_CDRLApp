## 📌 文档概述

本指南基于 CDRL 系统的 Word 文档解析器（`word_parser.py`）的实际代码实现，提供监督通知书的标准化格式要求。遵循本指南编写的通知书可以被系统 **100% 准确识别和解析**。

### 解析器核心特性

- **自动结构检测**: 支持二级结构和三级结构
- **智能信息提取**: 基于正则表达式和上下文分析
- **容错能力**: 支持多种标点符号和格式变体
- **上下文回填**: 自动补全缺失的施工/监理单位信息

---

## 🏗️ 文档结构类型

系统支持两种文档结构类型，会自动检测并适配：

### 1. 二级结构（标段 + 问题）

**特征**: 标段行中直接包含工点名称

**示例**:
```
（一）由中建八局施工、甘肃铁科监理的LWZQ-8标紫荆瑶山隧道出口（检查日期：2025年8月7日）
（1）问题描述...
（2）问题描述...
```

**检测规则**: 标段行包含工点特征词（桥、隧道、站、路基等）

### 2. 三级结构（标段 + 工点 + 问题）

**特征**: 标段行和工点行分开

**示例**:
```
（一）由中建八局施工、甘肃铁科监理的LWZQ-8标（检查日期：2025年8月7日）
1. 紫荆瑶山隧道出口
（1）问题描述...
（2）问题描述...
```

**检测规则**: 标段行不包含工点特征词

## ✅ 必填字段识别规范

### 1. 通知书编号

**位置**: 文档前 20 段内

**支持格式**:
```
✅ 南宁站〔2025〕（通知）玉岑08号
✅ 南宁站[2025]（通知）柳梧10号
✅ 宁建监2025-11
✅ 编号：南宁站〔2025〕（通知）黄百11号
```

**正则表达式**:
```python
r'南宁站[〔\[]2025[〕\]]\（通知\）\S+\d+号'
r'宁建监\d{4}-\d+'
r'编号[：:]\s*(\S+)'
```

**要求**:
- ✅ 使用全角或半角括号均可
- ✅ 年份必须为 4 位数字
- ✅ 编号必须唯一（系统会检测重复）
- ❌ 不要使用特殊字符（除了〔〕[]（）-）

---

### 2. 检查日期

**位置**: 文档前 30 段内

**标准格式**:
```
✅ 2025年8月7日
✅ 2025-08-07
```

**正则表达式**:
```python
r'(\d{4})[年-](\d{1,2})[月-](\d{1,2})[日]?'
```

**要求**:
- ✅ 年份必须为 4 位数字
- ✅ 月份和日期可以是 1-2 位数字（系统会自动补零）
- ✅ 支持"年月日"或"-"分隔符
- ❌ 不要使用"/"分隔符

---

### 3. 项目名称

**位置**: 第一段话中，"对"字之后

**标准格式**:
```
✅ 柳梧铁路
✅ 黄百铁路广西段
✅ 玉岑铁路
```

**识别规则**:
- 句子结构: `南宁监督站XXX对【项目名称】XXXX标...`
- 必须包含"铁路"两个字
- 在标段编号之前

**正则表达式**:
```python
r'([^，。；\s]+铁路[^，。；\s]*?)(?=[A-Z]{2}[A-Z]+(?:-?\d+)?标)'
```

---

### 4. 检查单位

**位置**: 第一段话中

**标准格式**:
```
✅ 南宁监督站
✅ 广西监督站
```

**识别规则**:
- 句子结构: `【检查单位】XXX对XXX铁路...`
- 必须包含"监督站"

**正则表达式**:
```python
r'([^，。；\s]+监督站)'
```

---

### 5. 检查人员

**位置**: 第一段话中，"监督站"之后

**标准格式**:
```
✅ 南宁监督站卢浩、陈胜根据...
✅ 南宁监督站唐小林、罗斌、蒋德义对...
```

**识别规则**:
- 位于"监督站"和"根据"/"按照"/"对"之间
- 人名用顿号（、）分隔
- 优先在"根据"或"按照"处截断

**要求**:
- ✅ 使用顿号（、）分隔多个人名
- ✅ 不要在人名中间加空格
- ❌ 不要使用逗号（，）分隔人名

---

### 6. 建设单位

**位置**: 通知书编号后的 1-3 行

**标准格式**:
```
✅ 柳州铁路工程建设指挥部
✅ 广西铁路投资集团有限公司
✅ 云桂铁路广西有限责任公司：
```

**识别规则**:
- 在编号行后的 1-3 行内
- 包含"指挥部"或"公司"
- 如果有冒号，只提取冒号前的部分

**要求**:
- ✅ 完整的单位全称
- ✅ 可以在末尾加冒号
- ❌ 不要使用简称

---

### 7. 检查依据

**位置**: 第一段话中，"根据"或"按照"之后

**标准格式**:
```
✅ 根据《国铁集团关于开展在建铁路桥梁施工安全隐患排查整治的紧急通知》(铁建设电〔2025〕44号)、
   《国家铁路局综合司关于开展铁路桥梁工程质量安全问题隐患排查治理的通知》（国铁综工程监函[2025]351号）等文件
```

**识别规则**:
- 从"根据"或"按照"开始
- 到"等文件"、"等规定"、"等要求"、"等通知"结束

**要求**:
- ✅ 使用书名号《》标注文件名称
- ✅ 使用括号标注文号
- ✅ 多个文件用顿号（、）分隔
- ✅ 以"等文件"、"等规定"等词语结尾

## 🔍 可选字段识别规范

### 1. 问题总数声明

**位置**: 文档前 20 段内

**标准格式**:
```
✅ 检查发现各类安全质量问题53个（安全问题12个、质量问题21个、管理行为问题20个）
✅ 共计发现各类问题79个，其中质量问题27个，安全问题29个，管理行为及其他问题23个
```

**正则表达式**:
```python
r'检查发现各类.*?问题(\d+)个'
r'(?:共计)?发现各类问题(\d+)个'
r'安全问题(\d+)个'
r'质量问题(\d+)个'
r'管理行为(?:及其[他它])?问题(\d+)个'
```

**要求**:
- ✅ 明确标注总数和分类数量
- ✅ 使用"个"作为单位
- ⚠️ 系统会验证声明数量与实际识别数量是否一致

## 📝 问题章节格式规范

### 章节结构

监督通知书通常包含以下章节：

```
一、检查基本情况
二、下发整改通知单的工点及问题
三、其他主要质量安全问题
四、监督意见和有关要求
```

**章节识别规则**:
- "二、" + ("下发整改通知单" 或 "不良行为" 或 "工点及问题") → 整改通知单章节
- "三、" + ("其他" 或 "其它" 或 "主要") → 其他问题章节
- "四、" 或 "监督意见" 或 "有关要求" → 结束标志

---

### 格式1：柳梧格式（有一级编号）

#### 下发整改通知单章节

**标准格式**:
```
二、下发整改通知单的工点及问题

（一）由中建八局施工、甘肃铁科监理的LWZQ-8标紫荆瑶山隧道出口（检查日期：2025年8月7日）
检查情况：隧道二衬混凝土存在蜂窝麻面现象，影响结构质量。
处理措施：责令施工单位立即整改，2025年8月15日前完成整改。

（二）由中铁五局施工、西南交大监理的LWZQ-6标藤县北站（检查日期：2025年8月8日）
检查情况：站台混凝土表面不平整，存在裂缝。
处理措施：责令施工单位进行修补，2025年8月20日前完成整改。
```

**识别规则**:
- 一级编号：`（一）`、`（二）`、`（三）` 等（全角括号 + 中文数字）
- 标段行格式：`（X）由【施工单位】施工、【监理单位】监理的【标段编号】标【工点名称】（检查日期：【日期】）`
- 问题描述：`检查情况：【问题描述】`
- 处理措施：`处理措施：【整改要求】`

---

```

（2）站房钢结构焊缝存在未焊透现象。
```

**识别规则**:
- 一级编号：`（一）`、`（二）` - 标段行
- 二级编号：`1.`、`2.` - 工点行
- 三级编号：`（1）`、`（2）` - 问题行

## 🔢 序号格式要求

### 一级编号（标段编号）

**标准格式**: `（一）`、`（二）`、`（三）`、`（四）`、`（五）`...

**要求**:
- ✅ **必须使用全角括号** `（）`
- ✅ **必须使用中文数字**: 一、二、三、四、五、六、七、八、九、十
- ❌ 不要使用半角括号 `()`
- ❌ 不要使用阿拉伯数字

**正则表达式**:
```python
r'^（[一二三四五六七八九十]）'
```

---

### 二级编号（工点编号或问题编号）

**标准格式**: `1.`、`2.`、`3.`...

**要求**:
- ✅ 使用阿拉伯数字
- ✅ **半角点（.）或全角点（．）均可**
- ✅ **顿号（、）也可以**
- ❌ 不要使用中文数字

**正则表达式**:
```python
r'^\d+[．、\.]'
```

**示例**:
```
✅ 1. 工点名称
✅ 1．工点名称
✅ 1、工点名称
❌ 一、工点名称
```

---

### 三级编号（问题编号）

**标准格式**: `（1）`、`（2）`、`（3）`...

**要求**:
- ✅ **必须使用全角括号** `（）`
- ✅ 使用阿拉伯数字
- ❌ 不要使用半角括号 `()`
- ❌ 不要使用中文数字

**正则表达式**:
```python
r'^（\d+）'
```

**示例**:
```
✅ （1）问题描述
✅ （2）问题描述
❌ (1)问题描述  ← 半角括号
❌ （一）问题描述  ← 中文数字
```

---

### 序号层级总结

| 层级 | 用途 | 格式 | 括号类型 | 数字类型 | 示例 |
|------|------|------|----------|----------|------|
| 一级 | 标段 | （X） | 全角 | 中文 | （一）、（二） |
| 二级 | 工点 | X. | 无括号 | 阿拉伯 | 1.、2. |
| 三级 | 问题 | （X） | 全角 | 阿拉伯 | （1）、（2） |

## 🔧 标段编号格式规范

### 标段编号格式

**标准格式**: `[项目代码][标段类型]-[编号]标`

**支持格式**:
```
✅ HBZQ-1标     ← 带连字符
✅ QFSG1标      ← 无连字符
✅ LWZQ-8标     ← 带连字符
✅ HBZF标       ← 纯字母
✅ YCZQ-4标     ← 带连字符
```

**正则表达式**:
```python
r'([A-Z]{2,}[A-Z0-9]*(?:-?\d+)?)(?=标)'
```

**要求**:
- ✅ 至少 2 个大写字母开头
- ✅ 可以包含数字
- ✅ 连字符（-）可选
- ✅ 必须以"标"字结尾
- ❌ 不要使用小写字母
- ❌ 不要使用中文字符（除了"标"）

---

### 标段行完整格式

**格式1（带"由"字）**:
```
（一）由【施工单位】施工、【监理单位】监理的【标段编号】标【工点名称】（检查日期：【日期】）
```

**格式2（不带"由"字）**:
```
（一）【施工单位】施工、【监理单位】监理的【标段编号】标【工点名称】（检查日期：【日期】）
```

**格式3（单位自带标段编号）**:
```
1. 【施工单位】【标段编号】标施工、【监理单位】【标段编号】标监理的【工点名称】（检查时间：【日期】）
```

**示例**:

## ⚠️ 常见格式问题及解决方案

### 问题1：序号括号类型错误

**错误示例**:
```
❌ (一)中铁五局施工...  ← 使用了半角括号
❌ (1)问题描述...       ← 使用了半角括号
```

**正确示例**:
```
✅ （一）中铁五局施工...  ← 全角括号
✅ （1）问题描述...       ← 全角括号
```

**解决方案**:
- 在 Word 中输入时，使用中文输入法输入括号
- 或者使用查找替换功能：将 `(` 替换为 `（`，将 `)` 替换为 `）`

---

### 问题2：标段编号格式不规范

**错误示例**:
```
❌ lwzq-8标        ← 使用了小写字母
❌ LWZQ8标         ← 缺少连字符（如果原本应该有）
❌ LWZQ - 8标      ← 连字符前后有空格
```

**正确示例**:
```
✅ LWZQ-8标
✅ LWZQ8标         ← 如果确实不需要连字符
```

**解决方案**:
- 统一使用大写字母
- 连字符前后不要加空格
- 保持项目内标段编号格式一致

---

### 问题3：检查时间标注不一致

**错误示例**:
```
❌ （检查时间2025年8月7日）      ← 缺少冒号
❌ （检查日期 2025年8月7日）     ← 冒号前有空格
❌ （检查时间：2025/8/7）        ← 使用了斜杠分隔符
```

**正确示例**:
```
✅ （检查时间：2025年8月7日）
✅ （检查日期：2025年8月7日）
✅ (检查时间：2025年8月7日)     ← 半角括号也可以
```

**解决方案**:
- 统一使用"检查时间"或"检查日期"
- 冒号可选，但推荐加上
- 日期格式必须是"YYYY年MM月DD日"

---

### 问题4：施工/监理单位分隔符错误

**错误示例**:
```
❌ 中铁五局施工，西南交大监理    ← 使用了逗号
❌ 中铁五局施工 西南交大监理      ← 缺少分隔符

**正确示例**:
```
✅ 中铁五局施工、西南交大监理    ← 使用顿号
✅ 中铁五局施工，西南交大监理    ← 逗号也可以识别
```

**解决方案**:
- 推荐使用顿号（、）
- 逗号（，）也可以被识别
- 不要省略分隔符

### 问题5：工点名称提取失败

**错误示例**:
```
❌ LWZQ-8标（检查日期：2025年8月7日）紫荆瑶山隧道出口

**解决方案**:

### 问题6：问题描述格式不规范

### 问题8：跨行格式识别失败

**错误示例**:

- 格式：`X.【工点名称】（检查时间：【日期】）`

### 问题9：问题总数不匹配

## 📚 附录：正则表达式速查表

### 通知书编号
```python
r'南宁站[〔\[]2025[〕\]]\（通知\）\S+\d+号'
r'宁建监\d{4}-\d+'
```

### 日期
```python
r'(\d{4})[年-](\d{1,2})[月-](\d{1,2})[日]?'
```

### 标段编号
```python
r'([A-Z]{2,}[A-Z0-9]*(?:-?\d+)?)(?=标)'
```

### 一级编号（标段）
```python
r'^（[一二三四五六七八九十]）'
```

### 二级编号（工点）
```python
r'^\d+[．、\.]'
```

### 三级编号（问题）
```python
r'^（\d+）'
```

### 施工单位
```python
r'由(.+?)施工'  # 带"由"字
r'^\d+[．、\.]\ s*([^、，]+?)施工'  # 数字编号行
```

### 监理单位
```python
r'[、，]([^、，]+?)监理'
```

### 工点名称
```python
r'标(.+?)（检查(?:时间|日期)[：:]?'
```
