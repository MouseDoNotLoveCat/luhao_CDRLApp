# 导入流程优化 - 快速参考

## 🎯 优化目标

简化导入流程，减少不必要的中间步骤，提升用户体验。

## 🔄 流程变化

### 优化前
```
通知书预览 → 问题列表 → 问题浏览 → 问题编辑 → 确认 → 导入
```

### 优化后
```
通知书预览 → 问题列表 → 问题编辑 → 确认 → 导入
```

**改进**：减少 1 个步骤

## ✅ 完成的修改

### 1. ImportPreviewNotices.vue
```javascript
// 直接跳转到编辑界面
importStore.viewMode = 'edit-issues'
```

### 2. ImportIssuesEditor.vue
- 移除问题编号列
- 返回到通知书预览界面
- 保存后直接进入确认界面

### 3. ImportConfirm.vue
- 显示 16 个问题字段
- 严重程度带颜色标签
- 返回到编辑界面

## 📊 表格字段对比

### 编辑界面
- 序号、标段、工点、问题描述
- 问题类别（三层级联）
- 严重程度

### 确认界面（优化后）
- 序号、标段、工点、问题描述
- 问题类别、问题子类1、问题子类2
- 严重程度（带颜色）
- 整改通知、不良行为（带标签）
- 检查单位、检查日期、检查人员
- 整改要求、整改期限
- 责任单位、责任人
